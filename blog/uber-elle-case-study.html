Building ELLE: Award-Winning AI Decision Engine at Uber

Executive Summary

The Challenge

Uber's privacy team processed 1,000+ data flow decisions manually, requiring expert review of each request against complex privacy policies spanning hundreds of documents. This created:

  • Bottlenecks: 3-5 day turnaround times for decisions
  • Inconsistency: Decisions varied based on reviewer availability and interpretation
  • Scalability issues: Manual process couldn't keep pace with Uber's growth
  • Expert dependency: Only senior privacy engineers could make complex decisions

The team needed an AI system that could:

  • Understand complex privacy policies (GDPR, CCPA, internal standards)
  • Provide reasoned decisions with explanations
  • Maintain human oversight (human-in-the-loop)
  • Scale to 1,000+ decisions daily
  • Integrate with existing workflow tools

The Solution

We built ELLE, an LLM-powered decision engine with a human-in-the-loop workflow combining:

Core Architecture

  • LangChain orchestration for complex multi-step reasoning
  • RAG system for policy document retrieval (1,000+ privacy documents)
  • React dashboard for privacy team review and override
  • GCP infrastructure (Cloud Functions, Cloud Run, Firestore)
  • Kafka integration for real-time data flow monitoring

Key Technical Decisions

1. Why LangChain?

Complex multi-step reasoning required orchestration of LLM calls, policy retrieval, and decision trees. LangChain provided the framework for managing this complexity while maintaining transparency.

2. Why RAG over Fine-Tuning?

Privacy policies change frequently (weekly updates). RAG allowed us to update the knowledge base dynamically without retraining models. Also provided transparency—we could trace exactly which policy sections influenced each decision.

3. Why Human-in-the-Loop?

High-stakes decisions require expert validation. ELLE provides AI-generated recommendations with reasoning, but privacy engineers make final calls. This hybrid approach combines AI speed with human judgment.

4. Why GCP?

Existing Uber infrastructure. Easy integration with internal tools, proven security posture, and strong SLA guarantees for production systems.

Lessons Learned

1. Start Simple, Iterate Fast

Shipped v1 in 4 weeks with basic policy matching. Refined based on privacy team feedback over next 8 weeks. Rapid iteration > perfect first version.

2. Human-in-the-Loop is Critical for High-Stakes Decisions

Initially considered fully automated decisions. Privacy team insisted on human oversight. This proved essential—AI provides speed and consistency, humans provide judgment and accountability.

3. RAG > Fine-Tuning for Dynamic Knowledge

Privacy policies updated weekly. RAG allowed instant updates to knowledge base without retraining. Also provided transparency—could trace which policies influenced decisions.

4. Prompt Engineering Matters

Spent 30% of development time on prompt optimization. Quality of LLM responses directly correlated with prompt clarity and structure. Invested in systematic prompt testing and refinement.

5. Monitor, Monitor, Monitor

Production LLM systems require extensive monitoring. Tracked decision latency, LLM API costs, error rates, and human override frequency. Alerts for anomalies prevented issues from escalating.
